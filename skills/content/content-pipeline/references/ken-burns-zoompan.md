# Ken Burns Zoompan — Pipeline v3

Реализация плавного зума/пана в Pipeline v3 через ffmpeg `zoompan` filter.

## Базовая формула

```
scale=2160:-2:force_original_aspect_ratio=increase,
crop=2160:3840,
zoompan=z='1.0+0.0003*on':d={seg_frames}:x='iw/2-(iw/zoom/2)':y='ih/2-(ih/zoom/2)':s=1080x1920:fps=30,
format=yuv420p
```

### Как это работает

1. **scale + crop** — апскейлим исходное 1080×1920 до 2160×3840 (2×). Нужно, чтобы при зуме не было пикселизации.
2. **zoompan** — создаёт N кадров (`d={seg_frames}`) с постепенным зумом, начиная с 1.0x.
3. **z выражение** — `1.0 + 0.0003 * on`. `on` = номер кадра (0-based). За ~335 кадров (11с при 30fps) зум = 1.0→1.1 (10%).
4. **x/y выражения** — центрируют изображение: `x = iw/2 - (iw/zoom/2)`. При зуме в 1.1x, x сдвигается на 5% влево (визуально центрирует зум).
5. **s=1080x1920** — выходное разрешение. zoompan сам ресайзит результат обратно.
6. **fps=30** — частота кадров.

### Выбор коэффициента зума

| seg_frames | coeff  | zoom end | визуальный эффект      |
|------------|--------|----------|------------------------|
| 300 (10с)  | 0.0003 | ×1.09    | незаметный, едва живёт |
| 300        | 0.0005 | ×1.15    | хороший, плавный       |
| 300        | 0.001  | ×1.30    | заметный, драматичный  |
| 100        | 0.002  | ×1.20    | резкий, дёрганый       |

**Рекомендация:** `0.0003` для сегментов >8 секунд, `0.0005` для 5–8 секунд.

## Чередование типов движения

```
if i % 2 == 0:  # чётные — zoom in
    zexpr = "1.0+0.0003*on"
    xexpr = "iw/2-(iw/zoom/2)"
    yexpr = "ih/2-(ih/zoom/2)"
else:           # нечётные — pan + лёгкий zoom
    zexpr = "1.0+0.0002*on"
    direction = 60 if i % 4 == 1 else -60
    xexpr = f"iw/2-(iw/zoom/2)+{direction}*sin(on/100)"
    yexpr = "ih/2-(ih/zoom/2)"
```

- **Pan** добавляется через `sin(on/100)` — плавное колебание на 60px влево/вправо
- Направление меняется: i%4==1 → +60px, i%3==1 → -60px

## Альтернативные подходы

### 1. PIL progressive crop (без ffmpeg)
Для каждого кадра генерировать N PNG с прогрессивным кропом вместо zoompan.
Плюсы: не зависит от ffmpeg. Минусы: в N раз больше PNG файлов, дольше генерация.

### 2. ffmpeg zoompan на всё видео целиком
Вместо per-segment, наложить zoompan поверх уже склеенного видео.
Проблема: concat склеивает в одно видео, zoompan поверх не даст per-segment разный тип движения.

### 3. moviepy zoom
`video.resize(lambda t: 1+0.02*t)` — но moviepy медленнее ffmpeg + нужен re-encode.

## Edge Cases

- **Смена кадра при zoompan+concat:** возможна задержка на 1 кадр между сегментами — ffmpeg корректно стыкует через concat, но начальный кадр нового сегмента может дропнуться. Решение: `-vsync drop` или добавить 1 дополнительный кадр перекрытия.
- **Быстрый зум + низкое разрешение:** если исходник <1080p, апскейл до 2160×3840 даст пиксельный шум. Лучше уменьшить coeff до 0.0001.
- **Portrait vs Landscape:** в pipeline всегда 1080×1920 (portrait). Для landscape (1920×1080) менять scale/crop на 3840×2160.
- **FFmpeg версия:** zoompan доступен в ffmpeg ≥ 4.0. На 7.0.2 работает без проблем.
