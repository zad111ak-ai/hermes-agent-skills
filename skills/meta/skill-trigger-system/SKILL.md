# Skill Trigger System — Операционная система для скиллов

## Философия

Вместо "я вспомнил что надо загрузить скилл" → **система сама чует когда что нужно**.

```
Триггер сработал → ХОБА → загрузил скилл → использовал → выгрузил
```

## Типы триггеров

### 🔴 Триггер 1: Context Threshold (объём контекста)

Слежу за размером текущей сессии. Когда переваливает за отметку — автоматом применяю скилл.

| Метка | Что делать | Какой скилл |
|-------|-----------|-------------|
| Сессия >40 сообщений | ⚠️ Контекст растёт | `session-hygiene-plus` — провести гигиену |
| Сессия >80K токенов | 🔴 Критично! | `session-hygiene-plus` — форсировать сжатие |
| >10 tool calls без execute_code | 🟡 Много одиночных вызовов | `context-economy` — проверить можно ли батчить |
| >3 ошибки подряд | 🔴 Застрял? | `stuck-detection` — проверить зацикливание |
| >2 неудачных попытки сделать одно и то же | 🟡 | `fail-fast` — сменить подход |
| Достиг 80% результата задачи | 🟢 | `early-termination` — можно останавливаться? |

### 🟡 Триггер 2: Behavioral (поведение пользователя)

По паттерну сообщения пользователя понимаю какой скилл нужен.

| Что пишет пользователь | Триггер | Скилл |
|-----------------------|---------|-------|
| "почему не работает" / "ошибка" / "сломалось" | ❗ | `systematic-debugging` |
| "сравни" / "какой лучше" / "выбери" | 🔄 | `decision-framework` |
| "сделай план" / "с чего начать" | 📋 | `writing-plans` или `dev-methodology` |
| "сколько стоит" / "цена на" / "мониторинг" | 💰 | `price-monitor` |
| "напиши пост" / "контент" / "канал" | 📢 | `money-making-agency` |
| "настрой" / "установи" / "включи" | ⚙️ | `hermes-agent` |
| "найди инфу" / "поищи" / "что там по" | 🔍 | `duckduckgo-search` (через delegate_task) |
| "сгенерируй картинку" / "сделай аву" | 🎨 | (image_gen напрямую) |
| "проанализируй" / "проверь" | ✅ | `reflection-qa` |
| "почини" / "баг" / "не пашет" | 🐛 | `systematic-debugging` |
| "создай скилл" / "запомни как делать" | 📝 | `hermes-agent-skill-authoring` |
| "срочно" / "быстро" / "на вчера" | ⚡ | Пропустить все скиллы — делать сразу |

### 🟢 Триггер 3: Temporal (временные)

По контексту и времени понимаю что навык нужен.

| Ситуация | Триггер | |
|----------|---------|-|
| Закончил задачу → собираюсь ответить | ✅ | `auto-qa-gates` — проверить качество |
| Ответ больше 2000 символов | 📏 | `agent-communication` — сжать |
| Нужно выбрать инструмент (terminal vs execute_code vs delegate_task) | 🤔 | `decision-framework` |
| Задача с 5+ шагами | 🏗️ | `dev-methodology` |
| Пишу ответ в формальном стиле | 😤 | `humanizer` — оживить |
| Начинаю новую тему (контекст переключился) | 🔄 | Сбросить старые скиллы |

### 🔵 Триггер 4: Result-Driven (по результату)

Смотрю на ЧТО я получил и решаю — нужен ли скилл для обработки.

| Результат | Триггер | |
|-----------|---------|-|
| Сложный JSON / много данных | 📊 | Загрузить `context-economy` — обработать через execute_code |
| subagent вернул suspicious summary | 🚨 | `reflection-qa` — проверить не наврал ли |
| Появилась новая эффективная техника | 💡 | `self-improvement` — сохранить в скилл |

## Исполнение (цикл Load → Use → Unload)

```
[Триггер сработал]
    ↓
ХОБА! — загружаю скилл (skill_view)
    ↓
Беру из него суть — 2-3 ключевых пункта
    ↓
Применяю к текущей ситуации
    ↓
СКИЛЛ ВЫГРУЖЕН — больше не висит в контексте
```

**Критическое правило:** Не хранить скилл в контексте дольше 1-2 ответов. Загрузил → взял инструкцию → применил → забыл.

## Пример работы (как это выглядит)

### Пример 1: Пользователь пишет "Почему не работает?"

```
1. [Триггер: behavioral — "почему не работает"]
2. ХОБА → skill_view('systematic-debugging')
3. Беру: "4 фазы: Understand → Isolate → Fix → Verify"
4. Применяю к ошибке пользователя
5. Скилл выгружен
6. Отвечаю по схеме из скилла
```

### Пример 2: Сессия разрослась до 45 сообщений

```
1. [Триггер: context threshold — >40 msgs]
2. ХОБА → skill_view('session-hygiene-plus')
3. Беру: "Держать ≤50K, execute_code вместо последовательных tool calls"
4. Переключаю режим на execute_code/delegate_task
5. Скилл выгружен
```

### Пример 3: Пишу ответ на сложный вопрос

```
1. [Триггер: temporal — закончил задачу]
2. ХОБА → skill_view('auto-qa-gates')
3. Проверяю 5 gates: Completeness → Correctness → Verifiability → Clarity → Cost
4. Всё ок → отвечаю
5. Скилл выгружен
```

## Priority System (что важнее)

Когда срабатывает несколько триггеров одновременно — приоритет:

```
🔴 CRITICAL (stuck-detection, fail-fast)
    ↓
🟡 IMPORTANT (session-hygiene, systematic-debugging)
    ↓
✅ NORMAL (reflection-qa, decision-framework)
    ↓
🟢 OPTIONAL (agent-communication, humanizer)
```

Одновременно можно обработать максимум 2 триггера. Если их больше — выполняй только Top-2 по приоритету.

---

## Anti-Patterns

- ❌ **Загрузить 5 скиллов сразу** → контекст раздулся, толку ноль
- ❌ **Держать скилл в контексте "на всякий случай"** → выгружай сразу после применения
- ❌ **Скилл внутри скилла** → загрузил A, он сказал загрузить B — стоп. Не делай цепочку
- ❌ **Триггер на каждое чихание** → не все сообщения требуют скилла. Вопрос "как дела?" → 0 скиллов
- ❌ **Загрузить скилл, прочитать целиком, выгрузить не применив** → бесполезная трата токенов
