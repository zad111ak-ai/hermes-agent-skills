# Примеры: ДО и ПОСЛЕ применения скиллов

Наглядные сравнения, которые показывают разницу.

---

## 📊 session-hygiene-plus: Экономия токенов

### ДО ❌
```
Агент: "Я вижу что предыдущее задание было связано с парсингом данных...
вот что я нашёл в контексте... давайте я продолжу с того места..."
[Потрачено: 15,000 токенов контекста]
[Стоимость: $0.45]
```

### ПОСЛЕ ✅
```
Агент: "Готово! Вот результаты парсинга."
[Потрачено: 2,000 токенов контекста]
[Стоимость: $0.06]
[Экономия: 87% — $0.39 сэкономлено]
```

**Как это работает:** session-hygiene-plus автоматически сжимает контекст и удаляет неактуальные данные. Агент не тратит токены на "вспоминание" того, что уже не нужно.

## 💰 cost-aware-orchestrator: Выбор модели

### ДО ❌
```
Задача: "Переведи этот текст на английский"
Модель: GPT-4 (3.00$ / 1M токенов)
Стоимость: $0.003
```

### ПОСЛЕ ✅
```
Задача: "Переведи этот текст на английский"
Модель: Groq Llama 3.1 8B (0.05$ / 1M токенов)
Стоимость: $0.00005
Экономия: 98%
```

**Как это работает:** cost-aware-orchestrator анализирует сложность задачи и автоматически выбирает самую дешёвую модель, которая справится. Простые задачи → дешёвые модели. Сложные → дорогие.

## 🔍 post-mortem-analyzer: Анализ ошибок

### ДО ❌
```
Агент: "Произошла ошибка. Попробуйте ещё раз."
[Агент не понимает причину ошибки]
[Повторяет ту же ошибку снова и снова]
```

### ПОСЛЕ ✅
```
Агент: "Ошибка связана с тем, что API вернул 429 (rate limit).
Причина: слишком частые запросы к одному эндпоинту.
Решение: добавить задержку 1.5с между запросами.
Предотвращение: использовать asyncio.Semaphore для ограничения параллелизма."
[Агент понимает и исправляет корневую причину]
```

## 🔒 security-audit: Находки

### ДО ❌
```python
# Захардкоженный ключ в коде
API_KEY = "sk-1234567890abcdef"
query = f"SELECT * FROM users WHERE name = '{user_input}'"
```

### ПОСЛЕ ✅
```python
# Ключ в .env
API_KEY = os.getenv("API_KEY")

# Параметризованный запрос
query = "SELECT * FROM users WHERE name = ?"
cursor.execute(query, (user_input,))
```

**Что нашёл security-audit:**
- ✅ Захардкоженный API-ключ
- ✅ SQL injection в строке 42
- ✅ Path traversal в строке 67
