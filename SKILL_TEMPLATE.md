# 📋 Шаблон скилла для Hermes Agent

Скопируй этот файл как основу для своего скилла.

---

## YAML-фронтмат (обязательно)

```yaml
---
name: my-skill-name
description: "Одно предложение: что делает скилл и зачем нужен."
tags: ["tag1", "tag2", "tag3"]
---
```

**Требования к name:**
- Латиница, дефисы, без пробелов
- Максимум 64 символа
- Пример: `cost-aware-orchestrator`, `security-audit`

**Требования к description:**
- Одно предложение
- Начинается с глагола или существительного
- Без "Этот скилл делает..." — сразу к делу

---

## Структура SKILL.md

```markdown
# Название скилла — Короткое описание

Активировать: когда [триггер].

**Цель:** [одно предложение о чём скилл].

---

## Зачем

[1-3 абзаца: какую проблему решает. Конкретика.]

---

## Как работает

[Логика скилла: что делает, в каком порядке.]

---

## Правила

1. **[ВСЕГДА/НИКОГДА]** правило
2. **[ВСЕГДА/НИКОГДА]** правило
3. **[ВСЕГДА/НИКОГДА]** правило

---

## Примеры

### Хорошо ✅
[Конкретный пример использования]

### Плохо ❌
[Конкретный пример неправильного использования]

---

## Метрики

- [Что измеряется]
- [Какая экономия/ускорение]
```

---

## Чек-лист перед отправкой PR

- [ ] YAML-фронтмат заполнен
- [ ] name — латиница, дефисы, ≤64 символа
- [ ] description — одно предложение
- [ ] tags — минимум 2 тега
- [ ] Есть раздел "Зачем" (минимум 2 абзаца)
- [ ] Есть раздел "Правила" (минимум 3 правила)
- [ ] Есть примеры (Хорошо ✅ / Плохо ❌)
- [ ] Нет захардкоженных ключей/API
- [ ] Нет опасных команд без предупреждений
- [ ] Описание на русском языке
